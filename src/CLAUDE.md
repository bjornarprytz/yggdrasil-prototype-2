# Coding Agent Instructions

When completing coding tasks, make changes directly and silently — no inline diffs, no before/after comparisons, no lengthy explanations. Once done, write at most one short paragraph summarizing what changed and flag any notable tradeoffs, surprises, or problems encountered in the approach. Then list the key files that were modified or created so the user can review the diff in their IDE. Do not walk through the code line by line or explain what each function does unless explicitly asked.
